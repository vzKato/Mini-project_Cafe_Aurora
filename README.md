### 📌 **Project Overview**
Café Aurora is a mini project developed for the Data Fundamentals course. The goal is to create, populate, and analyze a database using a synthetic dataset of 1,000 sales records from the past few months of Café Aurora, a coffee shop chain with multiple branches across different cities in Mexico.

This project applies essential concepts of database design, SQL querying, Python, and exploratory data analysis.

### 🎯 **Objectives**
The management team at Café Aurora wants to understand key business insights from recent sales. This project focuses on answering:

1. Which products are the top sellers?
Identify the items with the highest sales volume.
2. Which city generates the most revenue?
Compare the financial performance of each branch by location.
3. How do sales behave over time?
Analyze trends, seasonality, and monthly variations.

### 🛠️ **Technologies Used**

- Google Colab + Python + SQL
- Pandas (for data handling)

### 📊 **Expected Outcomes**
By completing this project, the following results are obtained:
- A ranking of the best‑selling products
- A revenue comparison across cities
- Time‑based sales analysis through tables or visualizations
- Data‑driven conclusions to support business decisions

### 📎 **Final Notes**
Although the dataset is synthetic, the workflow simulates real‑world data analysis in a multi‑branch coffee shop environment. This project reinforces skills in data modeling, cleaning, querying, and interpretation.
